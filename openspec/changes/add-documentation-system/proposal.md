# Change: Add Comprehensive Documentation System

## Why
Current documentation (README.md, Docs/user-guide.md) covers basic usage but lacks critical setup guides for MCP server integration, multi-AI tool configuration, git workflow conventions, and contribution guidelines. These gaps block adoption and make it difficult for users to leverage full Cortext capabilities.

## What Changes
- Add MCP server setup and usage documentation
- Add tool-specific configuration guides (OpenCode, Gemini CLI, Cursor)
- Document git workflow conventions (branch naming, commit format)
- Create contribution guide for developers
- Organize documentation into discoverable structure

## Impact
- Affected specs: NEW capability `documentation` (no existing spec)
- Affected code: `Docs/` directory structure
- No code changes required - documentation only
